# 📅 Módulo de Agendas - Documentação Completa\n\n## Visão Geral\n\nO módulo de **Agendas** é um sistema inteligente e completo de agendamento integrado ao OMNI BUILDER CRM. Ele vai muito além de um calendário simples, funcionando como uma **máquina de geração e conversão de receita** com recursos avançados de qualificação de leads, distribuição automática e integração profunda com o CRM.\n\n## 🎯 Objetivos do Sistema\n\n- ✅ Gerar leads automaticamente através de agendamentos\n- ✅ Qualificar leads com base em comportamento de agendamento\n- ✅ Distribuir leads inteligentemente entre etapas do CRM\n- ✅ Integração nativa com o CRM (sem dependências externas)\n- ✅ Preparar base para monetização futura\n- ✅ Reduzir taxa de no-show com confirmações automáticas\n- ✅ Aumentar conversão com UX gamificada\n\n## 🏗️ Arquitetura\n\n### Estrutura de Banco de Dados\n\n#### 1. **Schedules** (Agendas)\nArmazena as agendas principais com configurações gerais.\n\n```sql\n- id: UUID (PK)\n- user_id: UUID (FK - auth.users)\n- name: TEXT\n- description: TEXT\n- slug: TEXT (UNIQUE)\n- timezone: TEXT\n- default_duration_minutes: INTEGER\n- is_active: BOOLEAN\n- is_published: BOOLEAN\n- whatsapp_number: TEXT\n- whatsapp_message_template: TEXT\n- created_at: TIMESTAMP\n- updated_at: TIMESTAMP\n```\n\n#### 2. **Schedule Services** (Serviços)\nDefine os serviços disponíveis em cada agenda.\n\n```sql\n- id: UUID (PK)\n- user_id: UUID (FK)\n- schedule_id: UUID (FK)\n- name: TEXT\n- description: TEXT\n- duration_minutes: INTEGER\n- price: DECIMAL (opcional)\n- color: TEXT\n- pipeline_id: UUID (referência ao CRM)\n- stage_id: UUID (referência ao CRM)\n- sort_order: INTEGER\n- is_active: BOOLEAN\n```\n\n#### 3. **Appointments** (Agendamentos)\nRegistra todos os agendamentos realizados.\n\n```sql\n- id: UUID (PK)\n- user_id: UUID (FK)\n- schedule_id: UUID (FK)\n- service_id: UUID (FK)\n- lead_id: UUID (FK - leads, opcional)\n- client_name: TEXT\n- client_email: TEXT\n- client_phone: TEXT\n- appointment_date: TIMESTAMP\n- appointment_end: TIMESTAMP\n- status: TEXT (scheduled|confirmed|cancelled|no_show|completed)\n- notes: TEXT\n```\n\n#### 4. **Availability Rules** (Regras de Disponibilidade)\nDefine horários de funcionamento por dia da semana.\n\n```sql\n- id: UUID (PK)\n- schedule_id: UUID (FK)\n- day_of_week: INTEGER (0-6)\n- start_time: TIME\n- end_time: TIME\n- is_available: BOOLEAN\n```\n\n#### 5. **Capacity Rules** (Regras de Capacidade)\nControla limites de agendamentos.\n\n```sql\n- id: UUID (PK)\n- schedule_id: UUID (FK)\n- max_appointments_per_day: INTEGER\n- max_appointments_per_slot: INTEGER\n- buffer_minutes_between: INTEGER\n- allow_overbooking: BOOLEAN\n```\n\n#### 6. **Blocked Dates** (Datas Bloqueadas)\nBloqueia períodos específicos.\n\n```sql\n- id: UUID (PK)\n- schedule_id: UUID (FK)\n- start_date: DATE\n- end_date: DATE\n- reason: TEXT\n```\n\n#### 7. **Schedule Automations** (Automações)\nDefine ações automáticas baseadas em eventos.\n\n```sql\n- id: UUID (PK)\n- schedule_id: UUID (FK)\n- trigger_event: TEXT (appointment_created|confirmed|cancelled|no_show)\n- action_type: TEXT (create_lead|update_stage|create_task|update_status)\n- action_config: JSONB\n- is_active: BOOLEAN\n```\n\n#### 8. **Schedule Events** (Eventos do Sistema)\nLog de todos os eventos do sistema.\n\n```sql\n- id: UUID (PK)\n- schedule_id: UUID (FK)\n- appointment_id: UUID (FK, opcional)\n- event_type: TEXT\n- event_data: JSONB\n- created_at: TIMESTAMP\n```\n\n## 📁 Estrutura de Arquivos\n\n```\nsrc/\n├── components/admin/\n│   ├── AdminSchedules.tsx          # Listagem e criação de agendas\n│   ├── ScheduleDetails.tsx         # Detalhes, serviços e disponibilidade\n│   ├── ScheduleDashboard.tsx       # Analytics e métricas\n│   └── ScheduleAutomations.tsx     # Gerenciamento de automações\n├── pages/\n│   └── PublicSchedulePage.tsx      # Página pública de agendamento\n├── hooks/\n│   └── useSchedulesQuery.ts        # Hooks React Query para API\n├── types/\n│   └── schedules.ts                # Tipos TypeScript\n├── utils/\n│   └── scheduleUtils.ts            # Funções utilitárias\nsupabase/\n└── migrations/\n    └── 20260402000000_create_schedules_module.sql\n```\n\n## 🎨 Componentes Principais\n\n### 1. AdminSchedules\nTela principal de gerenciamento de agendas.\n\n**Funcionalidades:**\n- Listar todas as agendas do usuário\n- Criar nova agenda\n- Editar agenda existente\n- Deletar agenda\n- Publicar/despublicar agenda\n- Buscar agendas por nome\n\n### 2. ScheduleDetails\nTela de configuração detalhada de uma agenda.\n\n**Abas:**\n- **Serviços**: Criar, editar e deletar serviços\n- **Disponibilidade**: Configurar horários por dia da semana\n- **Capacidade**: Limites de agendamentos e buffers\n\n### 3. PublicSchedulePage\nFluxo público de agendamento (cliente final).\n\n**Passos:**\n1. Seleção de serviço\n2. Seleção de data\n3. Seleção de horário\n4. Preenchimento de dados (nome, email, telefone)\n5. Tela de confirmação gamificada com botão WhatsApp\n\n### 4. ScheduleDashboard\nDashboard com analytics e métricas.\n\n**Visualizações:**\n- KPIs (total, confirmados, taxa de comparecimento, no-show)\n- Gráfico de agendamentos por dia\n- Distribuição de status\n- Lista de agendas ativas\n\n### 5. ScheduleAutomations\nGerenciamento de automações baseadas em eventos.\n\n**Eventos Suportados:**\n- Agendamento criado\n- Agendamento confirmado\n- Agendamento cancelado\n- Não compareceu\n\n**Ações Disponíveis:**\n- Criar lead no CRM\n- Atualizar etapa do lead\n- Criar tarefa\n- Atualizar status\n\n## 🔌 Hooks React Query\n\n### Schedules\n```typescript\nuseSchedules()                    // Listar todas as agendas\nuseScheduleBySlug(slug)          // Buscar agenda por slug\nuseScheduleById(id)              // Buscar agenda por ID\nuseCreateSchedule()              // Criar agenda\nuseUpdateSchedule()              // Atualizar agenda\nuseDeleteSchedule()              // Deletar agenda\n```\n\n### Services\n```typescript\nuseScheduleServices(scheduleId)  // Listar serviços\nuseCreateScheduleService()       // Criar serviço\nuseUpdateScheduleService()       // Atualizar serviço\nuseDeleteScheduleService()       // Deletar serviço\n```\n\n### Appointments\n```typescript\nuseAppointments(scheduleId)      // Listar agendamentos\nuseCreateAppointment()           // Criar agendamento\nuseUpdateAppointment()           // Atualizar agendamento\n```\n\n### Availability & Capacity\n```typescript\nuseAvailabilityRules(scheduleId) // Regras de disponibilidade\nuseUpdateAvailabilityRules()     // Atualizar disponibilidade\nuseCapacityRules(scheduleId)     // Regras de capacidade\nuseUpdateCapacityRules()         // Atualizar capacidade\n```\n\n### Blocked Dates\n```typescript\nuseBlockedDates(scheduleId)      // Datas bloqueadas\nuseCreateBlockedDate()           // Bloquear data\nuseDeleteBlockedDate()           // Desbloquear data\n```\n\n### Automations\n```typescript\nuseScheduleAutomations(scheduleId)    // Listar automações\nuseCreateScheduleAutomation()         // Criar automação\nuseUpdateScheduleAutomation()         // Atualizar automação\nuseDeleteScheduleAutomation()         // Deletar automação\n```\n\n## 🛠️ Funções Utilitárias\n\n### generateAvailableSlots\nGera slots disponíveis para uma data específica.\n\n```typescript\nconst slots = generateAvailableSlots(\n  date,\n  schedule,\n  service,\n  availabilityRules,\n  capacityRules,\n  blockedDates,\n  existingAppointments\n);\n```\n\n### generateSlug\nGera slug a partir do nome da agenda.\n\n```typescript\nconst slug = generateSlug(\"Consultas Médicas\");\n// Resultado: \"consultas-medicas\"\n```\n\n### formatAppointmentDateTime\nFormata data e hora para exibição.\n\n```typescript\nconst formatted = formatAppointmentDateTime(\"2026-04-15T14:30:00\");\n// Resultado: \"15 de abril de 2026 às 14:30\"\n```\n\n### generateWhatsAppLink\nGera link para WhatsApp com mensagem pré-preenchida.\n\n```typescript\nconst link = generateWhatsAppLink(\n  phoneNumber,\n  clientName,\n  serviceName,\n  appointmentDate,\n  appointmentTime,\n  messageTemplate\n);\n```\n\n### calculateScheduleAnalytics\nCalcula métricas de uma agenda.\n\n```typescript\nconst analytics = calculateScheduleAnalytics(appointments, services);\n// Retorna: {\n//   total_appointments: 50,\n//   confirmed_appointments: 45,\n//   attendance_rate: 90,\n//   ...\n// }\n```\n\n## 🔐 Segurança e RLS\n\nTodas as tabelas possuem **Row Level Security (RLS)** habilitado:\n\n- Usuários só veem seus próprios dados\n- Agendas publicadas podem ser acessadas publicamente via slug\n- Agendamentos públicos podem ser criados sem autenticação\n- Todas as operações respeitam `user_id` do usuário autenticado\n\n## 📱 Fluxo Público\n\n### URL Pública\n```\n/agenda/{slug}\n```\n\n### Passos do Cliente\n1. Acessa `/agenda/consultas-medicas`\n2. Seleciona um serviço\n3. Escolhe uma data (próximos 30 dias)\n4. Seleciona um horário disponível\n5. Preenche seus dados (nome, email, telefone)\n6. Confirma agendamento\n7. Vê tela de sucesso com animação\n8. Pode compartilhar no WhatsApp\n\n## 🤖 Sistema de Automações\n\n### Exemplo: Criar Lead ao Agendar\n```typescript\n{\n  trigger_event: \"appointment_created\",\n  action_type: \"create_lead\",\n
