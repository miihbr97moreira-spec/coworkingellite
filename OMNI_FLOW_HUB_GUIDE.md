# Omni Flow Hub & Omni Checkout - Guia de Implementação\n\n## 📋 Índice\n\n1. [Visão Geral](#visão-geral)\n2. [Omni Flow Hub](#omni-flow-hub)\n3. [Integração com CRM](#integração-com-crm)\n\n---\n\n## 🎯 Visão Geral\n\nEste projeto implementa dois componentes principais para automação de vendas e gerenciamento de leads:\n\n### **Omni Flow Hub**\nHub central de automações visuais com React Flow que permite criar fluxos automáticos usando:\n- **Gatilhos (Triggers)**: Lead Movido, Webhook Recebido, Formulário Enviado, Pagamento Recebido\n- **Ações (Actions)**: Mensagem WhatsApp, Agente IA, Atualização CRM, Email, Webhook, Delay\n\n### **Omni Checkout**\nComponente de pagamento transparente que:\n- Processa pagamentos de forma segura\n- Move leads automaticamente para \"Fechado/Ganho\" no CRM\n- Registra transações e envia confirmações\n- Integra-se perfeitamente com landing pages\n\n---\n\n## 🚀 Omni Flow Hub\n\n### Tipos de Gatilhos\n\n#### 1. **Lead Movido** (`lead_moved`)\nDispara quando um lead muda de estágio no CRM.\n\n```typescript\nconst trigger: LeadMovedTrigger = {\n  id: 'trigger_1',\n  type: 'lead_moved',\n  name: 'Lead Movido para Proposta',\n  description: 'Dispara quando lead vai para estágio Proposta',\n  config: {\n    toStage: 'proposal',\n    fromStage: 'qualified'\n  },\n  enabled: true\n};\n```\n\n#### 2. **Webhook Recebido** (`webhook_received`)\nDispara quando um webhook externo é recebido.\n\n```typescript\nconst trigger: WebhookReceivedTrigger = {\n  id: 'trigger_2',\n  type: 'webhook_received',\n  name: 'Novo Contato via Webhook',\n  description: 'Recebe novos contatos de sistema externo',\n  config: {\n    webhookUrl: '/api/webhooks/contacts',\n    method: 'POST',\n    eventType: 'contact.created'\n  },\n  enabled: true\n};\n```\n\n#### 3. **Formulário Enviado** (`form_submitted`)\nDispara quando um formulário é submetido.\n\n```typescript\nconst trigger: FormSubmittedTrigger = {\n  id: 'trigger_3',\n  type: 'form_submitted',\n  name: 'Formulário de Contato',\n  description: 'Dispara quando formulário é preenchido',\n  config: {\n    formId: 'contact_form_001',\n    formName: 'Contato'\n  },\n  enabled: true\n};\n```\n\n#### 4. **Pagamento Recebido** (`payment_received`)\nDispara quando um pagamento é confirmado.\n\n```typescript\nconst trigger: PaymentReceivedTrigger = {\n  id: 'trigger_4',\n  type: 'payment_received',\n  name: 'Pagamento Confirmado',\n  description: 'Dispara quando pagamento é processado',\n  config: {\n    minAmount: 100,\n    maxAmount: 10000,\n    paymentMethod: 'credit_card'\n  },\n  enabled: true\n};\n```\n\n### Tipos de Ações\n\n#### 1. **Mensagem WhatsApp** (`whatsapp_message`)\nEnvia mensagem automática via WhatsApp.\n\n```typescript\nconst action: WhatsAppAction = {\n  id: 'action_1',\n  type: 'whatsapp_message',\n  name: 'Enviar Boas-vindas',\n  description: 'Envia mensagem de boas-vindas',\n  config: {\n    phoneField: 'phone',\n    messageTemplate: 'Olá {{name}}, bem-vindo! 🎉',\n    variables: { name: 'lead.name' }\n  },\n  enabled: true\n};\n```\n\n#### 2. **Agente IA** (`ai_agent`)\nExecuta um agente de IA para processar dados.\n\n```typescript\nconst action: AIAgentAction = {\n  id: 'action_2',\n  type: 'ai_agent',\n  name: 'Análise de Lead',\n  description: 'Analisa qualidade do lead com IA',\n  config: {\n    agentId: 'agent_001',\n    agentName: 'Lead Analyzer',\n    prompt: 'Analise este lead e determine a probabilidade de conversão',\n    model: 'gpt-4.1-mini',\n    temperature: 0.7\n  },\n  enabled: true\n};\n```\n\n#### 3. **Atualizar CRM** (`crm_update`)\nAtualiza dados do lead no CRM.\n\n```typescript\nconst action: CRMUpdateAction = {\n  id: 'action_3',\n  type: 'crm_update',\n  name: 'Mover para Fechado/Ganho',\n  description: 'Move lead para estágio final',\n  config: {\n    leadId: 'lead_123',\n    stage: 'closed_won',\n    fields: { status: 'customer' },\n    tags: ['paid', 'customer']\n  },\n  enabled: true\n};\n```\n\n#### 4. **Email** (`email`)\nEnvia email automático.\n\n```typescript\nconst action: EmailAction = {\n  id: 'action_4',\n  type: 'email',\n  name: 'Confirmação de Compra',\n  description: 'Envia email de confirmação',\n  config: {\n    emailField: 'email',\n    subject: 'Compra Confirmada - {{productName}}',\n    template: '<h1>Obrigado pela compra!</h1>',\n    variables: { productName: 'product.name' }\n  },\n  enabled: true\n};\n```\n\n#### 5. **Webhook** (`webhook_call`)\nChama um webhook externo.\n\n```typescript\nconst action: WebhookCallAction = {\n  id: 'action_5',\n  type: 'webhook_call',\n  name: 'Notificar Sistema Externo',\n  description: 'Notifica sistema externo sobre conversão',\n  config: {\n    webhookUrl: 'https://api.external.com/notify',\n    method: 'POST',\n    headers: { 'Authorization': 'Bearer token' },\n    body: { event: 'lead_converted' }\n  },\n  enabled: true\n};\n```\n\n#### 6. **Delay** (`delay`)\nAguarda um tempo especificado.\n\n```typescript\nconst action: DelayAction = {\n  id: 'action_6',\n  type: 'delay',\n  name: 'Aguardar 24 Horas',\n  description: 'Aguarda 24 horas antes da próxima ação',\n  config: {\n    delayMs: 24,\n    delayType: 'hours'\n  },\n  enabled: true\n};\n```\n\n---\n\n## 💳 Omni Checkout Integrado\n\n### Configuração Básica\n\n```typescript\nimport { OmniCheckoutIntegrated, OmniCheckoutConfig } from '@/components/builder/OmniCheckoutIntegrated';\n\nconst checkoutConfig: OmniCheckoutConfig = {\n  id: 'checkout_001',\n  type: 'checkout',\n  productName: 'Curso Premium',\n  productValue: 297.00,\n  currency: 'BRL',\n  description: 'Acesso completo ao curso com certificado',\n  successRedirectUrl: 'https://seu-site.com/obrigado',\n  errorRedirectUrl: 'https://seu-site.com/erro'\n};\n\n<OmniCheckoutIntegrated\n  config={checkoutConfig}\n  onSuccess={(data) => {\n    console.log('Pagamento confirmado!', data);\n    // data.leadId - ID do lead criado/atualizado\n    // data.productName - Nome do produto\n    // data.productValue - Valor pago\n  }}\n  onError={(error) => {\n    console.error('Erro no pagamento:', error);\n  }}\n/>\n```\n\n### Fluxo de Processamento\n\n1. **Validação de Dados**: Valida todos os campos do formulário\n2. **Processamento de Pagamento**: Processa o pagamento com gateway\n3. **Criação/Atualização de Lead**: Cria novo lead ou atualiza existente\n4. **Movimentação de Estágio**: Move lead para \"Fechado/Ganho\"\n5. **Registro de Transação**: Registra a transação no banco de dados\n6. **Envio de Email**: Envia confirmação por email\n\n### Integração com CRM\n\nO Omni Checkout automaticamente:\n\n- ✅ Cria novo lead se não existir (por email)\n- ✅ Atualiza dados do lead existente\n- ✅ Move lead para estágio \"closed_won\"\n- ✅ Adiciona tags: `['paid', 'customer', productName]`\n- ✅ Registra valor e data da compra\n- ✅ Incrementa contador de compras\n\n---\n\n## 🏗️ Arquitetura\n\n### Estrutura de Arquivos\n\n```\nsrc/\n├── types/\n│   └── omniflow.ts                 # Tipos e interfaces\n├── services/\n│   ├── flowService.ts              # Serviço de fluxos\n│   └── checkoutService.ts          # Serviço de checkout\n├── components/\n│   ├── admin/\n│   │   └── OmniFlow/\n│   │       ├── OmniFlowHub.tsx      # Hub principal\n│   │       ├── FlowNodeTypes.tsx    # Tipos de nós\n│   │       ├── FlowPanel.tsx        # Painel de elementos\n│   │       ├── OmniFlowAgent.tsx\n│   │       ├── OmniFlowWebhooks.tsx\n│   │       └── OmniFlowWhatsApp.tsx\n│   └── builder/\n│       └── OmniCheckoutIntegrated.tsx # Checkout integrado\n└── pages/\n    ├── AutomationFlows.tsx         # Gerenciamento de fluxos\n    └── CheckoutDashboard.tsx       # Dashboard de vendas\n```\n\n### Fluxo de Dados\n\n```\nGatilho (Trigger)\n    ↓\n[FlowService.executeFlow()]\n    ↓\nIteração de Nós\n    ↓\nExecutar Ações em Sequência\n    ↓\n[FlowService.executeAction()]\n    ↓\nAtualizar CRM / Enviar Mensagens / etc\n    ↓\nRegistrar Execução\n```\n\n---\n\n## 📖 Guia de Uso\n\n### Criar um Fluxo de Automação\n\n1. Acesse **Fluxos de Automação**\n2. Clique em **Novo Fluxo**\n3. Nomeie o fluxo (ex: \"Onboarding de Clientes\")\n4. Clique em **Criar Fluxo**\n5. No editor visual:\n   - Arraste um **Gatilho** do painel esquerdo\n   - Arraste **Ações** para o canvas\n   - Conecte-os com linhas\n6. Clique em **Salvar Fluxo**\n7. Clique em **Executar** para testar\n\n### Exemplo: Fluxo de Bem-vindo\n\n```\nGatilho: Lead Movido → \"Qualified\"\n    ↓\nAção 1: Enviar WhatsApp (\"Olá {{name}}, bem-vindo!\")\n    ↓\nAção 2: Delay (1 hora)\n    ↓\nAção 3: Enviar Email (\"Conheça nossos serviços\")\n    ↓\nAção 4: Atualizar CRM (tag: \"contacted\")\n```\n\n### Adicionar Checkout a uma Landing Page\n\n```tsx\nimport { OmniCheckoutIntegrated } from '@/components/builder/OmniCheckoutIntegrated';\n\nexport function MyLandingPage() {\n  return (\n    <div>\n      <h1>Meu Produto Premium</h1>\n      <p>Descrição do produto...</p>\n      \n      <OmniCheckoutIntegrated\n        config={{\n          id: 'checkout_lp_001',\n          type: 'checkout',\n          productName: 'Produto Premium',\n          productValue: 297.00,\n          description: 'Acesso vitalício com suporte',\n          successRedirectUrl: '/obrigado'\n        }}\n        onSuccess={(data) => {\n          console.log('Venda realizada:', data.leadId);\n        }}\n      />\n    </div>\n  );\n}\n```\n\n---\n\n## 🔗 Integração com CRM\n\n### Tabelas Necessárias no Supabase\n\n#### `automation_flows`\n```sql\nCREATE TABLE automation_flows (\n  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),\n  tenant_id UUID NOT NULL,\n  name VARCHAR NOT NULL,\n  description TEXT,\n  nodes JSONB NOT NULL DEFAULT '[]',\n  edges JSONB NOT NULL DEFAULT '[]',\n  enabled BOOLEAN DEFAULT true,\n  created_at TIMESTAMP DEFAULT NOW(),\n  updated_at TIMESTAMP DEFAULT NOW()\n);\n```\n\n#### `flow_executions`\n```sql\nCREATE TABLE flow_executions (\n  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),\n  flow_id UUID NOT NULL REFERENCES automation_flows(id),\n  trigger_id VARCHAR,\n  status VARCHAR NOT NULL,\n  started_at TIMESTAMP,\n  completed_at TIMESTAMP,\n  executed_actions JSONB,\n  lead_data JSONB,\n  created_at TIMESTAMP DEFAULT NOW()\n);\n```\n\n#### `transactions`\n```sql\nCREATE TABLE transactions (\n  id UUID PRIMARY KEY DEFAULT uuid_generate_v4(),\n  lead_id UUID NOT NULL REFERENCES crm_leads(id),\n  payment_id VARCHAR NOT NULL,\n  amount DECIMAL(10, 2) NOT NULL,\n  product_name VARCHAR,\n  status VARCHAR,\n  customer_name VARCHAR,\n  customer_email VARCHAR,\n  customer_phone VARCHAR,\n  processed_at TIMESTAMP,\n  created_at TIMESTAMP DEFAULT NOW()\n);\n```\n\n### Atualizar CRM Lead\n\nO Omni Checkout atualiza automaticamente:\n\n```sql\nUPDATE crm_leads SET\n  stage = 'closed_won',\n  status = 'customer',\n  tags = array_append(tags, 'paid'),\n  last_purchase_value = 297.00,\n  last_purchase_date = NOW(),\n  purchase_count = purchase_count + 1,\n  closed_at = NOW(),\n  updated_at = NOW()\nWHERE id = 'lead_id';\n```\n\n---\n\n## 🔐 Segurança\n\n- ✅ Validação de dados no execução\n\n### Pagamento falha\n- Valide dados do cartão\n- Confirme configuração do gateway de pagamento\n- Verifique logs de transação\n\n### Lead não atualiza no CRM\n- Confirme se o lead existe ou foi criado\n- Verifique permissões de acesso ao banco de dados\n- Valide estrutura dos dados\n\n---\n\n## 📞 Suporte\n\nPara dúvidas ou problemas, consulte:\n- Documentação: `/docs`\n- Issues:
